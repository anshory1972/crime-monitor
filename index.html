■ Motivation

Economic and financial distress are well-established drivers of crime. When households face sustained income shocks, rising unemployment, or acute cost-of-living pressures, the incidence of property crime, violent crime, and social unrest tends to rise. This relationship has been documented across both developed and developing economies and operates through multiple channels: reduced legitimate earning opportunities, breakdown of community cohesion, and heightened desperation among vulnerable populations.

Conversely, a measurable and sustained increase in crime — particularly in regions with limited safety nets — can serve as a leading or coincident indicator that a population is under serious financial stress, even before that distress is captured by lagging official statistics such as poverty rates or household consumption surveys.

The Dewan Ekonomi Nasional (DEN), as the National Economic Council of Indonesia, advises the President on strategic economic policy and is mandated to monitor economic conditions and welfare across the archipelago. Crime trend monitoring falls squarely within that mandate: it provides DEN analysts with a timely, granular, and independent signal of financial distress at the district and provincial level — a signal that complements traditional macroeconomic indicators and may surface emerging vulnerabilities earlier.

This dashboard operationalises that monitoring function using near-real-time global event data from GDELT, delivering weekly crime intensity trends across all 34 provinces and 500+ districts of Indonesia.

■ Methodology
1. Data Source: GDELT

The Global Database of Events, Language and Tone (GDELT) is the world's largest open event dataset. Created by Kalev Leetaru (Georgetown University) and Philip Schrodt, it monitors print, broadcast and web news media in over 100 languages across every country, updated every 15 minutes.

GDELT automatically identifies real-world events — protests, violence, diplomacy, arrests — from news articles using machine learning and natural language processing, then codes them using the CAMEO (Conflict and Mediation Event Observations) taxonomy of over 300 event types. Each event is georeferenced to a location and linked to the number of news sources that reported it.

This dashboard queries the GDELT v2 Events table (gdelt-bq.gdeltv2.events) hosted as a public dataset on Google BigQuery, covering records from 2024 onwards.

2. Event Selection

Events are filtered to Indonesia only (ActionGeo_CountryCode = 'ID') and to CAMEO codes associated with crime-proximate behaviour:

Code prefixCategory
14xProtests & demonstrations
18xAssault & physical violence
19xMass violence & coercion

A 24-month rolling window is maintained, pulling approximately 66,000 raw event records before cleaning. After deduplication, roughly 27,000 unique geo-coded events remain, spanning 34 provinces and 383 districts.

↻ Data updates: The pipeline is re-run periodically to pull the latest GDELT events. The dashboard header shows the date range of the current dataset; check it to confirm how recent the data are.

3. Crime Intensity Index

A raw event count treats a single-source local incident the same as a nationally covered atrocity. To correct for this, each event is weighted by two GDELT fields:

intensity = NumMentions × |GoldsteinScale|
  • NumMentions — how many news sources worldwide reported the event; a proxy for significance and scale.
  • GoldsteinScale — a theoretical conflict severity score from −10 (most destabilising) to +10 (most cooperative). Crime-proximate events cluster near −10.
4. Administrative Boundaries

District-level attribution is performed via spatial join using the official BPS (Badan Pusat Statistik) administrative boundary shapefile at Admin Level 2 (kabupaten/kota), distributed by OCHA's Humanitarian Data Exchange (HDX, accessed 2024).

The boundary file covers 522 kabupaten/kota across 34 provinces in WGS84 (EPSG:4326) projection — the same coordinate system used by GDELT's ActionGeo_Lat/Long fields, requiring no reprojection. Each GDELT event point is assigned to the containing district polygon; approximately 2.5% of points falling in coastal or offshore positions are retained with a null district assignment.

5. Data Pipeline
  1. BigQuery pull — SQL query against gdelt-bq.gdeltv2.events using a service account key; results downloaded as a Parquet file.
  2. Cleaning — null coordinate removal, bounding box validation, deduplication (same date/code/location, keep highest NumSources).
  3. Spatial join — GeoPandas point-in-polygon against BPS adm2 boundaries to assign province and district.
  4. Weekly aggregation — Monday-anchored ISO weeks; intensity, event count, and mean Goldstein computed per week × province.
  5. Static export — JSON files generated by Python and served as a fully static HTML dashboard (Chart.js + Leaflet.js), deployable to GitHub Pages.
6. Limitations & Caveats
  • Media bias — GDELT measures news coverage, not actual crime incidence. Events in high-media-density areas (Jakarta, Java) will appear more intense than equivalent events in low-coverage regions.
  • Under-reporting — Rural, remote, and small-scale incidents that do not reach regional or national news outlets will be absent from the data.
  • CAMEO coding accuracy — Event classification is automated; some records may be miscoded or represent non-criminal conflict.
  • Temporal lag — The pipeline runs on demand; data reflect the last export date, not real-time.
  • Indicator, not measure — This dashboard is a monitoring and early-warning tool. Findings should be validated against official BPS crime statistics and field intelligence before informing policy decisions.

⚠ Disclaimer & Known Limitations

This dashboard is in active development. Data, calculations, and visualisations may contain errors or inaccuracies. It should not be used as the sole basis for policy decisions without independent verification against official sources.

Methodological weaknesses to bear in mind:

  • Media coverage, not official crime records — Data are derived from news articles indexed by GDELT, not from police reports, BPS crime statistics, or court records. Events that receive no media coverage are entirely absent.
  • Java / urban bias — Regions with higher journalist density and internet penetration (Jakarta, Surabaya, Bandung) will appear more active regardless of actual crime rates, systematically understating conditions in remote provinces.
  • Automated event coding — CAMEO classification is performed by machine-learning models. Miscoding, duplicate events from multiple articles about the same incident (partially mitigated by deduplication), and misgeolocation are inherent risks.
  • Spatial precision — GDELT geolocates events to named places; precision varies from district level to province level. Approximately 2.5% of events cannot be matched to a district polygon and are excluded from district-level charts.
  • No causal inference — Correlation between crime intensity and economic distress is well-supported in the literature, but this dashboard cannot establish causation for any specific event or trend.
  • Static data snapshot — The dashboard reflects data as of the last pipeline run. It is not updated in real time; the data range shown in the header indicates the latest available date.
